# Documentation

SurakshaNet AI documentation is split by responsibility:

* `Product/` for scope, backlog, and release planning.
* `Business/` for pilot and value strategy.
* `Architecture/` for system, API, and database design.
* `AI/` for verification rules and model upgrade planning.
* `Security/` for privacy and user-protection guidance.
* `QA/` for test planning and acceptance coverage.
* `Agents/` for role-specific AI-agent instructions.
* `Management/` for roadmap, workflow, blockers, and execution status.
