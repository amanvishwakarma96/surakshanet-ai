# MVP Scope

## MVP Goal

Deliver a practical civic safety workflow where citizens can report hazards, receive nearby alerts, see verified public issues, and request limited community help while preserving privacy and auditability.

## In Scope

* Incident reporting for flood, electric hazard, pothole/road hazard, unsafe area, and general civic safety reports.
* Media and location metadata planning, with approximate location shown publicly.
* MVP verification status flow: submitted, needs review, verified, rejected, published, resolved.
* Mock AI scoring guidance for severity, department, radius, and sensitive flagging.
* Geo-fenced alert planning for nearby users.
* Public accountability board for verified issues only.
* Solution suggestions for practical non-legal civic action.
* Verified helper requests using approximate location first.
* Audit log requirements for sensitive actions.

## Out of Scope for MVP Setup

* Flutter implementation.
* .NET implementation.
* SQL schema implementation.
* Production AI model integration.
* Payment, petition filing, and legal-aid automation.
* Real-time dispatch integrations.

## MVP Success Criteria

* Future tasks can be assigned by module and role.
* Each sensitive workflow has a documented privacy expectation.
* Backend, database, mobile, QA, security, and documentation work can proceed independently from clear starter documents.
